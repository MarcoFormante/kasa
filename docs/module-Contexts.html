(static) AlertHandlerContext

Global Notification System

A centralized context provider that manages the visibility, content, and duration of the global Alert system. It allows any descendant component to trigger notifications without prop drilling.

Core Functionality:

  • State Management: Controls the visibility toggle, message content, and severity type (success, error, info) of the active alert.
  • Top-Level Rendering: Integrates the Alert component at the root level to ensure it overlays the entire application UI.
  • Auto-Dismiss Logic: Handles the lifecycle of notifications, including automatic closing after a specified timeout.

Technical Note: This provider should be placed high in the component tree (usually within the Root Layout) to be accessible by all application routes.

(static) FavoritesContext

Favorites State & Persistence Manager

A centralized context provider responsible for the lifecycle of the user's favorite property listings. it ensures that the favorites data is consistent across the UI and persistent between sessions.

Core Operations:

  • State Initialization: On component mount, it retrieves the initial favorites list from the server using the getFavorites utility.
  • LocalStorage Persistence: Automatically synchronizes the active state with the browser's local storage to ensure quick access and offline availability.
  • Data Synchronization: Provides an optimized interface for child components to toggle favorites without manual prop-drilling or redundant API calls.

Technical Note: This provider utilizes a useEffect hook to monitor state changes and trigger side effects for data persistence.
